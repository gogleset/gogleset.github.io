# 자료와 컴퓨터
## 자료 압축
- 자료를 저장하는데 필요한 공간을 줄이는 것

## 압축률
-  자료가 압축된 후의 크기를 압축되기 전의 원래의 크기로 나눈 비율

## 자료 압축 기술
1. 비손실 압축 : 압축을 하는 과정에서 잃어버리는 정보 없이 원래 자료와 동일하게 복원되는 압축기술
2. 손실 압출 : 압축을 하는 과정에서 약간의 정보를 잃어버리는 압축 기술

### 텍스트 압축기술
-  문서와 각 문자에 16비트를 할당하면 너무 많은 파일공간을 차지한다. 문서를 효과적으로 저장하고 전송하는 방법이 필요했다.

#### 1. 키워드 압축
- 자주 사용하는 단어를 한 개의 문자로 대체(압축률이 어떤 문장을 쓰느냐에 따라 다르다.)

#### 2. 실행 길이 압축
- 하나의 문자가 매우 길게 반복해서 사용되는 경우가 있다.
- 반복되는 문자열을 다음으로 대체한다.
  - 지시(flag)문자
  - 반복되는 문자
  - 반복 횟수
- **ex)** *n8
  - *는 지시문자
  - n은 반복되는 문자
  - 8은 반복횟수
- **ex)** 구체적 변환 예시
  ```
  bbbbbbbbbbjjjkiiqqqqqq+++++
  *b9jjjkii*q6*+5
  ```
###### 왜 예시에서 jjjkii는 압축되지 않나요? 짜피 byte는 똑같기 때문에 디코딩이나 인코딩할때 효율적이지 않다.

- 이 압축방식은 동영상, 그래픽 등에서 효과적으로 쓰일 수 있다.
#### 호프만 압축
- 자주 사용되는 문자를 비트로 표현하는 방식
- 8bit로 표현하는 문자들을 0과1로 표현해 2,4bit로 표현이 가능하다.

## 아날로그 정보와 디지털 정보
- 정보는 아날로그와 디지털 중 하나의 방식으로 표현될 수 있다.
- 아날로그 자료 : 실제정보와 같이 연속적 방식으로 표현
- 디지털 자료 : 정보를 작게 나누어서 비연속적인 방식으로 표현(주기적으로 디지털신호는 원래의 형태를 회복하기 위해 복원된다.)

## 텍스트의 표현
- 텍스트를 표현해야 하는데 제공해야 하는 것은 표현해야 하는 문자의 수는 유한하므로 모든 문자들을 나열하고, 각 문자들에 대해서 이진 bit열(bit string)을 할당한다.

### 문자의 집합
- 문자들과 각 문자들을 나타내기 위해 정의돈 코드들의 리스트

#### ASCII 문자 
- 하나의 문자가 1byte
- ASCII는 정보교환 목적의 미국 표준코드(American Standard Code for Information Interchange)의 약자

![ASCII](/assets/img/ASCII.png){: width="400" height="400"}

- ASCII는 원래 각 문자에 7bit씩 사용하여 128개의 고유 문자를 표현한다. 나중에 확장형 ASCII가 나와 8bit를 모두 사용하게 되었다.


#### Unicode 문자
- 2byte를 사용하여 전세계 모든 문자를 표현가능하다.
- 하나의 Unicode 매핑에는 문자 한개당 16비트를 사용한다.
- Unicode는 ASCII의 상위 집합이고 Unicode의 처음 256문자는 확장형 ASCII 문자와 정확히 일치한다.
